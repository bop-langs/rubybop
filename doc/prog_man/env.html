BOP Environment Variables

BOP uses two environment variables as follows. They can be set in a shell window or in the parallelized program.

BOP_GroupSize

The environment variable controls how many processors a BOP execution should use. By default, it is set to 2, which means 2 PPR tasks are executed in parallel each time. To set the variable to a new value, e.g. 8, use the following command in a Bash shell

BOP_GroupSize=8;export BOP_GroupSize

BOP_Verbose

The environment variable controls how much diagnosis output to generate during an execution. To suppress all internal output, set the variable to 0 using the following command in a Bash shell

BOP_Verbose=0;export BOP_Verbose

A positive verbose level shows the parallel execution in increasing amount of details depending on the level. Here is short summary:

  • BOP_Verbose = 1 prints out information about task creation and completion, conflicts in shared data access if there is any, and the status of sequential execution (called the understudy). It generates a report at the end of the execution.
  • BOP_Verbose = 2 prints out information about the execution of ordered blocks and the internal signaling before and after a PPR task.
  • BOP_Verbose = 6 prints out all information BOP tracks internally including the location and time of shared data access when recorded by BOP_record_read/write.

The C Interface

The following calls can be made from the parallelized program to change the value of the two variables. For example:

#include 
...
BOP_set_group_size(8);
BOP_set_verbose(0);
